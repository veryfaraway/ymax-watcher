# [PRD v2.0] Yong-IMAX Watcher (다중 극장 & 터보 모드)

> **문서 버전**: 2.0  
> **상태**: Draft  

## 1. 개요
* **목표**: 기존 v1.0의 단일 용아맥 감시를 넘어, 다중 특수관 감시, 취켓팅 모니터링, 임박 시 터보 모드, 오픈 통계 수집 기능을 안전하게 통합한다.

## 2. 주요 변경 사항
### F-1. 다중 극장 & 다중 특수관 지원 (Multi-Targets)
* `config.json`의 `targets` 배열을 순회하며 크롤링.
* 지원 hall_type: `IMAX`, `4DX`, `SCREENX`, `DOLBY` 등 텍스트 매칭 기반.

### F-2. 잔여 좌석(취켓팅) 모니터링
* 예매가 이미 오픈되었더라도, `available_seats`가 0석에서 1석 이상으로 증가하면 "취소표 발생" 알림 발송.
* 반복 알림(스팸) 방지: 좌석이 0석으로 떨어졌을 때만 상태를 리셋.

### F-3. 터보 모드 (동적 스케줄링) & Anti-Ban
* GHA Cron: `*/15 23,0-14 * * *` (15분 주기 실행)
* **일반 모드**: target date가 7일보다 많이 남은 경우, 매시 00~14분 사이에만 크롤링 실행 (사실상 1시간 주기).
* **터보 모드**: target date가 7일 이내인 target이 하나라도 있으면 매 15분마다 크롤링.
* **Rate Limit**: 각 HTTP 요청 간 `3~7초` 랜덤 딜레이 적용.

### F-4. 통계 기록 (history.json)
* 예매 오픈 감지 시 `history.json`에 Append 하여 이력 자산화.
